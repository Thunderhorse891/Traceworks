Launch Readiness Checklist

Use this page to confirm what is still needed before go-live.

What we need from you

  • Stripe live API keys + webhook secret.
  • SMTP production credentials and approved sender domain.
  • Owner/support email inbox for report copies and escalations.
  • Final legal policy language sign-off (Terms/Privacy/No-refund redo policy).
  • Admin API key + status token secret (long random values).

Operational checks

  • Scheduled worker is enabled and running.
  • Dead-letter monitoring route is documented.
  • Redo workflow owner is assigned.
  • At least one successful test-mode checkout completed end-to-end.
  • Inbox deliverability verified for customer + owner copies.

GitHub push readiness

If git push fails with “No configured push destination”, add and verify your repo remote first:

  • git remote add origin https://github.com/<owner>/<repo>.git
  • git push -u origin work (replace work with your active branch if different)
  • git remote -v should show origin fetch/push URLs.
